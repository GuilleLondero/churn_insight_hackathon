# churn_insight_hackathon
El desafío es crear una solución que prediga si un cliente es propenso a cancelar un servicio (churn).  El objetivo es que el equipo de Data Science desarrolle un modelo predictivo y que el equipo de Back-end construya una API para disponibilizar esa predicción a otros sistemas, permitiendo que el negocio actúe antes de que el cliente decida irse.
